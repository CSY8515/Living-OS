# Living OS Release Plan

## 목적

Living OS의 버전 관리와 릴리즈 기준을 정의한다.

모든 기능은 안정성과 검증을 우선하며,
완성되지 않은 기능은 다음 버전으로 이월한다.

# Version Policy

## v0.x

- 구조 구축
- 핵심 기능 개발
- 설계 검증
- 안정성 확보

## v1.0

- Stable Release
- 핵심 기능 완성
- 장기 운영 시작

---

# Release Rules

- 설계 없이 구현하지 않는다.
- 문서와 코드가 일치해야 한다.
- 테스트를 통과한 기능만 릴리즈한다.
- 변경 사항은 반드시 CHANGELOG에 기록한다.

---

# Release Checklist

- Roadmap 확인
- Master Blueprint 확인
- Architecture 확인
- Module Specs 확인
- 구현 완료
- 테스트 완료
- CHANGELOG 작성
- Git Commit
- Git Push
- Release
