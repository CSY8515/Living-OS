# Living OS Release Plan

## Version policy

- v0.x established the architecture and feature baseline.
- v1.0 Stable begins the production-stability line with the complete v0.9 feature set preserved.

## Release flow

Planning → implementation → regression testing → compatibility verification → Streamlit all-page smoke testing → documentation review → user approval → Git and release actions.

## Release rules

- Release only approved scope that matches the architecture and documentation.
- Do not release when regression, compatibility, AI-safety, startup, or page verification fails.
- Record notable changes in `CHANGELOG.md` and limitations in `KNOWN_ISSUES.md`.
- Exclude credentials, private user data, backups, temporary files, and generated bytecode from release artifacts.
- Never commit, push, tag, deploy, or publish without explicit user approval.

## v1.0 Stable checklist

- Approved v1.0 roadmap reviewed
- All v0.9 pages and data interfaces preserved
- JSON/JSONL schemas unchanged
- Regression and backward-compatibility tests pass
- AI remains optional, user-triggered, and read-only by default
- Compilation/import checks pass
- Headless Streamlit startup succeeds
- Every page passes a UI smoke check
- Current version labels and documentation agree
- Known limitations are documented
- Final diff contains no broad feature or architecture change
- Separate approval obtained before any Git or release action
